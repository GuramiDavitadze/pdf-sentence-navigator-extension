# PDF Sentence Navigator

Chrome Extension (Manifest V3), რომელიც საშუალებას გაძლევთ PDF დოკუმენტში
წინადადებებში გადაადგილდეთ კლავიატურით:

- **Tab** — შემდეგ წინადადებაზე გადასვლა
- **Shift + Tab** — წინა წინადადებაზე დაბრუნება

აქტიური წინადადება ვიზუალურად მონიშნულია (ყვითელი ხაზგასმა) პირდაპირ
დოკუმენტში.

ტექსტი მთლიანად მუშავდება **ლოკალურად, ბრაუზერში** — არაფერი არ იგზავნება
გარე სერვერზე. PDF-ის რენდერისთვის და ტექსტის ამოღებისთვის გამოყენებულია
[PDF.js](https://mozilla.github.io/pdf.js/) ბიბლიოთეკა, რომელიც ჩაშენებულია
extension-ში (`lib/` საქაღალდე) — ინტერნეტიდან არაფერი არ იტვირთება runtime-ზე.

## რატომ აქვს extension-ს "საკუთარი" PDF ვიუერი

Chrome-ის ჩაშენებულ PDF ვიუერს (PDFium) extension-ის content script-ები
უსაფრთხოების მიზეზების გამო ვერ სწვდება — ის იზოლირებულია და მისი
ტექსტ-ლეიერის მანიპულირება გარედან შეუძლებელია. ამიტომ ეს extension
იყენებს საკუთარ გვერდს (`viewer.html`), სადაც დოკუმენტი თავიდან
რენდერდება PDF.js-ით — სრული კონტროლით ტექსტზე, მონიშვნასა და ნავიგაციაზე.

## პროექტის სტრუქტურა

ყველა ფაილი ერთ დონეზეა (root-ში), ქვესაქაღალდეების გარეშე, გარდა `lib/`-ისა:

**მნიშვნელოვანია:** ყველა ფაილის ბილიკი ერთმანეთთან ფარდობითია
(`viewer.html` → `viewer.js`, `viewer.js` → `./lib/...`). თუ რომელიმე
ფაილს გადაიტანთ ქვესაქაღალდეში (მაგ. `src/`), ბმულები გაწყდება.

## `lib/` საქაღალდის მომზადება (PDF.js ბიბლიოთეკა)

`pdf.min.mjs` და `pdf.worker.min.mjs` ავტომატურად გენერირებული ფაილებია
(pdf.js ბიბლიოთეკის ნაწილი), რომლებიც თავად უნდა ჩამოტვირთოთ:

```bash
mkdir -p pdf-sentence-navigator/lib
npm install pdfjs-dist@4.10.38 --prefix /tmp/pdfjs-tmp
cp /tmp/pdfjs-tmp/node_modules/pdfjs-dist/build/pdf.min.mjs pdf-sentence-navigator/lib/
cp /tmp/pdfjs-tmp/node_modules/pdfjs-dist/build/pdf.worker.min.mjs pdf-sentence-navigator/lib/
```

## როგორ ჩავტვირთოთ extension Chrome-ში

1. გახსენით `chrome://extensions` მისამართი Chrome-ში.
2. ჩართეთ **Developer mode** (გადამრთველი ზედა მარჯვენა კუთხეში).
3. დააჭირეთ **Load unpacked**-ს.
4. აირჩიეთ პროექტის root საქაღალდე (სადაც `manifest.json` მდებარეობს).
5. Extension გამოჩნდება სიაში სახელით **"PDF Sentence Navigator"**.
6. თუ Chrome შეცდომას გამოსახავს — წაიკითხეთ ზუსტი ტექსტი
   (`chrome://extensions` → extension-ის ბარათი → **Errors**), ის
   ჩვეულებრივ მიუთითებს ზუსტად რომელი ფაილი აკლია ან სად არის პრობლემა.

## როგორ შემოწმდეს ფუნქციონალი

1. დააჭირეთ extension-ის icon-ს Chrome-ის ხელსაწყოთა პანელზე.
   გაიხსნება ახალი ტაბი extension-ის საკუთარი ვიუერით (`viewer.html`).
2. დააჭირეთ ღილაკს **"📂 PDF ფაილის გახსნა"** და აირჩიეთ ნებისმიერი PDF
   ფაილი თქვენი კომპიუტერიდან.
3. დოკუმენტი დარენდერდება გვერდზე, ხოლო ზედა პანელში გამოჩნდება ნაპოვნი
   წინადადებების რაოდენობა.
4. დააჭირეთ **Tab**-ს — მოინიშნება პირველი წინადადება (ყვითლად) და გვერდი
   ავტომატურად ჩაასქროლავს მასთან.
5. გააგრძელეთ **Tab**-ის დაჭერას — მონიშვნა გადავა შემდეგ წინადადებებზე.
6. დააჭირეთ **Shift + Tab**-ს — მონიშვნა დაბრუნდება წინა წინადადებაზე.
7. ბოლო წინადადებიდან Tab გადადის პირველზე თავიდან (და პირიქითაც) —
   ციკლური ნავიგაცია.

თუ რამე არ მუშაობს — გახსენით DevTools (`F12`) viewer.html-ის ტაბზე,
გადადით **Console**-ზე და შეამოწმეთ წითელი შეცდომები.

## როგორ მუშაობს წინადადებებად დაყოფა

`viewer.js` PDF.js-ის `TextLayer`-ის საშუალებით თითოეულ გვერდზე ტექსტის
თითოეულ ფრაგმენტს (`span`) აქცევს DOM ელემენტად, ზუსტად პოზიციონირებულს
დოკუმენტში არსებულ ტექსტზე. `sentence-navigator.js` შემდეგ ამ span-ებს
დოკუმენტის კითხვის თანმიმდევრობით (გვერდი-გვერდზე) ჯგუფავს
წინადადებებად — span-ები თანმიმდევრულად გროვდება "ურნაში", სანამ არ
მოხვდება ტექსტი, რომელიც მთავრდება `.`, `!`, `?` ან `…` სიმბოლოთი
(შესაძლოა ბრჭყალის/ფრჩხილის მიყოლებით) — ამ მომენტში ურნა იხურება,
როგორც ერთი დასრულებული წინადადება.

მონიშვნისას (`active-sentence` CSS კლასი) მთელი ჯგუფის ყველა `span`-ს
ემატება ყვითელი ფონი — ასე მიიღწევა ვიზუალური მონიშვნა თვითონ PDF
დოკუმენტში, თანაც სწორ ადგილას, მრავალხაზიან წინადადებებზეც კი.

## ცნობილი შეზღუდვები

- წინადადებების ამოცნობა ეფუძნება მარტივ პუნქტუაციურ ევრისტიკას
  (`.`, `!`, `?`, `…`). შემოკლებები (მაგ. "გვ." ან "Mr.") ან ათწილადი
  რიცხვები (მაგ. "3.14") შეიძლება არასწორად ჩაითვალოს წინადადების
  დასასრულად.
- სკანირებული (მხოლოდ სურათებისგან შემდგარი, OCR-გარეშე) PDF-ები
  ტექსტის ლეიერს არ შეიცავს — მათზე წინადადებების ნავიგაცია ვერ
  იმუშავებს, რადგან ამოსაღები ტექსტი უბრალოდ არ არსებობს დოკუმენტში.
- Extension ამჟამად მუშაობს მხოლოდ ხელით არჩეულ PDF ფაილებთან
  (extension-ის საკუთარი ვიუერის საშუალებით) — არ ხდება ავტომატური
  ჩარევა PDF ბმულების პირდაპირ ბრაუზერში გახსნისას.

## გამოყენებული ტექნოლოგიები

- Chrome Extension **Manifest V3** (`background` service worker, `chrome.action`)

- უბრალო JavaScript (ES modules), framework-ებისა და build-ნაბიჯის გარეშე
